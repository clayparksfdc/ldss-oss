Grid utility classes help create responsive layouts. These classes use the CSS flexbox model to align and distribute elements to ensure a consistent and scalable layout structure across Salesforce applications.

Examples

Basic Grid

Our most commonly used grid has standard 16px gutters. If you don’t have a design that calls for something more specific, this is the grid example you’ll want to use.

  1. Add a grid container by adding slds-grid to an HTML element
  2. Add as many slds-col elements as you want inside of your grid container

Initially, each column takes an equal percentage of the width available. In this case, that's 33.3% of the container’s width. Once content is added, the sections expand as needed to fit the content if sizing classes are not used.

Adding a fourth column means they'll each take up 25% of the width of the container.

Column Gutters

By default, grid columns don't have gutters. This allows flexibility to add and customize gutter widths to your liking for each column or grid container.

Modifying Gutters with Padding

You can also add different gutter widths to your columns by adding a padding utility (eg.slds-*p*-[*location]*_[*size])*to the slds-col element. The *[location]* can be left, right, top, bottom, vertical, horizontal, or around. The *[size]* can be xxx-small, x-small, small, medium, large, x-large, xx-large. For example, use slds-p-left_xxx-small for a 2px gutter on the left.

Class Definition

slds-p-right-*[size]* Add padding to the right
slds-p-top-*[size]* Add padding to the top
slds-p-bottom-*[size]* Add padding to the bottom
slds-p-vertical-*[size]* Add padding to top and bottom of each row
slds-p-horizontal-*[size]* Add padding to left and right of each column
slds-p-around-*[size]* Add padding to top, bottom, left, and right of a cell

This example adds gutters of 16px on each side (a total of 32px) in between each column.

Adding 16px of space on both sides of every column adds 16px of padding on the left and right of your container. To remove this unwanted space, add a slds-grid_pull-padded-[*size]* class to the slds-grid container. For the *[size]* portion of this class, use the same spacing utility size you've applied to the outermost columns.

Column Wrapping

Columns align horizontally unless you tell them to do something else. If you're using size classes (for example slds-size_4-of-12) and the total number of columns in a row exceeds 12, you can tell the grid container to wrap your columns to a new row. Just add slds-wrap to the slds-grid container.

Column Reordering

ℹ️ Accessibility Concern

If you change the order of columns without changing their position in the markup, your page can be hard to read for people using technology like screen readers. To stay WCAG compliant, make sure the visual order of columns matches the order in the markup.

Column Nesting

Nested columns give you more flexibility in your designs. To create nested columns, add another slds-grid container inside your column. Then, add as many slds-col elements inside that container as you want.

Grid Flow

With the slds-grid class, you can control the flow of your grid. By default, a grid flows from left to right on the horizontal axis. Changing it to flow from top to bottom on the vertical axis is as easy as adding slds-grid_vertical to the slds-grid container.

Left to Right

The default flow is left to right on the horizontal axis.

Top to Bottom

To switch the grid to flow top to bottom on the vertical axis, add slds-grid_vertical to the slds-grid container.

Reversed

Some situations call for reversing the visual flow of your columns. You can do this both horizontally and vertically.

Right to Left

To reverse the horizontal flow, add the class slds-grid_reverse to the slds-grid container.

Bottom to Top

To reverse the vertical flow, add the class slds-grid_vertical-reverse to the slds-grid container.

Creating Responsive Layouts

SLDS sizing and ordering utilities use a mobile-first approach, and support responsive breakpoints that change the layout at certain viewport sizes. The SLDS grid system becomes responsive to the viewport size when you specify responsive sizing and ordering utility classes. These classes include a breakpoint name in the SLDS size and order classes, for example slds-medium-size and slds-medium-order.

Add responsive sizing utility classes to an element, making sure they're in the correct order. Each class overrides the previous breakpoint, so list them from smallest to largest viewport.

The first sizing utility is applied at the smallest viewport width and continues to be applied until a responsive sizing utility for a specific size breakpoint overrides it.

In some rare cases, you may need a sizing utility that works up to a specified breakpoint. In those cases, use the classes that begin with max- to achieve your goal.

Conditional Column Reordering

With flexbox, you can rearrange your columns visually. The SLDS grid system makes it easy to reorder the columns visually in a responsive way. To do this, use the same naming convention as sizing. Insert the breakpoint name before order in slds-order, and then add an integer to indicate where you want the column to appear visually. The column with the lowest value of slds-order class appears first in the column order. If you don't define an order, the standard order is applied. This means that a column without an order class renders before slds-order_1. The following example sets the first column to be displayed last in the order of 3. Then when the viewport exceeds a width of 1024px, it changes the visual order of the first column back to 1.

Alignment

Because the grids are built on flexbox, they enable you to do some interesting things with alignment on both a horizontal axis and a vertical axis. You can add an alignment utility to the slds-grid container.

Horizontal Axis

Content centered

If you want your columns to grow from the center of the horizontal axis, apply the class slds-grid_align-center.

Setting variable widths based on content

Another powerful technique is letting a column shrink to the width of the content inside of it, while explicitly setting widths for other columns. This is achieved by using the slds-shrink-none or slds-grow-none utility classes or simply removing the slds-col class from the column element.

Content evenly spaced out

To evenly distribute columns on the horizontal axis with an equal amount of white space separating the columns, apply the class slds-grid_align-space.

Content evenly spread out from edges

To spread out your columns on the horizontal axis, with the first column starting at the start of your main axis and last item ending at the far end of your main axis, apply the class slds-grid_align-spread.

Content right-justified

If you want your columns to grow from the end of the horizontal axis, apply the class slds-grid_align-end.

Content alignment with bump

To "bump" a single or multiple grid items, apply the class slds-col_bump-{direction}, with {direction} being either left, right, top or bottom, to a grid item.

Vertical Axis

ℹ️ Note

To vertically align elements on a cross-axis of a slds-grid container, the elements need available vertical white space. You can usually achieve this by applying a height to the slds-grid container.

Content top

To align a single row or multi-line rows to the beginning of the cross axis, apply the class slds-grid_vertical-align-start.

Content centered

To vertically center align a single row or multi-line rows to the height of a slds-grid container, apply the class slds-grid_vertical-align-center. When you use slds-grid_vertical-align-center with slds-grid_align-center, the effect is to horizontally and vertically center align your content in the center of the slds-grid container.

Content bottom

To align a single row or multi-line rows to the end of the cross axis, apply the class slds-grid_vertical-align-end.

Content absolutely centered

Individual alignment

To specify the vertical placement of grid items on the cross axis, you can apply slds-align-top, slds-align-middle, and slds-align-bottom to an slds-col element.

Page Containers

Optionally, you can contain your page content by adding the class slds-container_*width* to a top-level element in your page structure.

Containers are available in four widths: small, medium, large, and x-large. They are rendered using max-widthto use the maximum width available to display their content. This means the content expands to fill the space until it reaches the width limit set for the container by slds-container_*.

Class Width
slds-container_small 480px
slds-container_medium 768px
slds-container_large 1024px
slds-container_x-large 1280px

Additionally, you can center the content of your page by adding slds-container_center.
