<p>Agentic experiences are systems and interfaces that empower users to accomplish their goals with the help of autonomous AI agents. Use these patterns to create agentic experiences in Salesforce products.</p>
<h2>AI-Enabled Cards</h2>
<p>Use AI-enabled cards for evergreen AI content within a page. Cards summarize and recommend actions after users generate content, regenerate content, and sometimes automatically generate. AI cards are compact and scannable, while still allowing users to expand AI insights if needed.</p>
<p>An empty agentic cardDoDo show an empty state until a user generates content.An agentic card with a data table and multiple actionsDon’tDon't display complex data on a single card.</p>
<p>Do'sDon'tsShow an empty state until a user generates content.Don't display complex data on a single card.Always show generated content sources.Avoid cluttering dashboards with too many AI-driven cards.Allow for human feedback.Stack cards if more than one is needed.</p>
<h2>Assistant Panels</h2>
<p>Assistant panels are optional conversational AI dialogs that users can open to get help and close when the help is no longer needed. Assistant panels facilitate a back-and-forth conversation with predictive and proactive interactions. Users can toggle the visibility of an assistant when needed. Additionally, assistant panels can resize, float, and be pinned above content. When closed, an assistant panel isn't visible but retains its content history and previous state. The prompt composer is docked at the bottom of the assistant panel.</p>
<p>DoDo allow panels in builders to pop out and float in order to provide optimal workspace.Don’tDon’t update data without human consent from the assistant panel.</p>
<p>Do'sDon'tsProvide prompt recommendations based on context and actions.Don’t update data without human consent from the assistant panel.Dock panels within modal dialogs.Don’t automatically open or close the panel without explicit action by the user.Allow panels in builders to pop out and float in order to provide optimal workspace.</p>
<h2>Canvases</h2>
<p>Use plain language to ask AI for a custom view of your information. Canvases are fully interactive and customizable areas of content generated by AI. They're also multimodal, personalized, and dynamic. Central to the canvas is a bottom-centered prompt composer.</p>
<p>DoDo display individual cards for each insight requested by the userDoUse multiple input methods besides text, such as voice.</p>
<p>Do'sDon'tsDisplay individual cards for each insight requested by the userDon’t replace existing UI elements with AI-generated components without user controlProvide actionable insights in each generated componentDon’t use text as the only allowed input methodUse signifiers to indicate the generated components can be moved and resized</p>
<h2>Disambiguation</h2>
<p>Disambiguation provides details and further grounding to AI, which turns vague prompts into data-specific requests.</p>
<p>DoProvide radio button groups with data-grounding options.Don’tDon’t make disambiguation a multi-step process.</p>
<p>Do'sDon'tsProvide radio button groups with data-grounding options.Don’t display checkboxes or text inputs to support disambiguation.Display an action to submit the disambiguation form.Don’t make disambiguation a multistep process.</p>
<h2>Feeds</h2>
<p>Actions and prompts appear chronologically in structured feeds. AI and human responses are clearly identified using avatars. The feed history is saved with an option to clear it. Additionally, structured data is presented in cards.</p>
<p>DoDo display feedback on each action.Don’tDon't fill the feed with long text and unnecessary AI responses.</p>
<p>Do'sDon'tsDisplay timestamps and authorship labels for actions.Don’t create dead ends in conversations.Display feedback on each action.Don't fill the feed with long text and unnecessary AI responses.Allow users to refine actions.</p>
<h2>Forms</h2>
<h3>AI-Enabled Form Fields</h3>
<p>AI can automatically generate form content and provide predictive input suggestions without disrupting manual input. Make sure that users retain control over autofill values in AI-assisted forms.</p>
<p>DoDo indicate AI-enabled fields with a single sparkle icon.DoAlways provide modification controls. Don’t restrict inputs to only generated data without enabling manual overrides.</p>

<h3>Form Dirty State Management</h3>
<p>When form fields have unsaved changes, they're in a dirty state. Fields in a dirty state from AI-generated changes are visually and programmatically marked as pending until confirmed. Additionally, AI-generated changes are visually distinct from human-created changes.</p>
<p>DoDo display change indicators for pending AI actions.Don’tAvoid silent AI modifications to critical business records.</p>

<h2>Human Feedback</h2>
<p>Let users share feedback on AI responses. Simple thumbs-up and thumbs-down options are quick. Users can also add detailed feedback in a long-form text input to provide more precise training data.</p>
