<h2>Overview</h2>
<ul>
<li>Checkbox toggle enables you to switch settings between two binary states, such as enabling or disabling features.</li>
<li>It’s commonly used for user preferences, feature settings, or workflows.</li>
<li>It's easy to use. You get instant feedback, so you know right away if a feature is on or off.</li>
</ul>
<h2>Usage</h2>
<p>The checkbox toggle is ideally suited for selecting one of two mutually exclusive states, such as activating or deactivating a feature. It’s commonly used in setting up Salesforce products, allowing users to consider their preferences or actions, such as enabling notifications or adjusting workflow automation settings.</p>
<h3>Best Practices</h3>
<ul>
<li>Always label the toggle so the user understands the action they’re taking.</li>
<li>Use toggles for settings that take effect immediately.</li>
<li>When a toggle is activated, give users immediate feedback to show if the action was successful.</li>
<li>Make the component accessible for keyboard navigation and screen readers.</li>
</ul>
<h3>When Not to Use</h3>
<ul>
<li>For actions involving more than two options, use other methods instead of a toggle switch.</li>
<li>Avoid using toggles for actions that require confirmation or have significant consequences.</li>
<li>Don’t use toggles to make a selection that is better suited for radio buttons, checkboxes, or picklists. </li>
<li>Don’t use toggles for irreversible actions (for example, deleting records).</li>
</ul>

## Overview

* Checkbox toggle enables you to switch settings between two binary states, such as enabling or disabling features.
* It’s commonly used for user preferences, feature settings, or workflows.
* It's easy to use. You get instant feedback, so you know right away if a feature is on or off.

## Usage

The checkbox toggle is ideally suited for selecting one of two mutually exclusive states, such as activating or deactivating a feature. It’s commonly used in setting up Salesforce products, allowing users to consider their preferences or actions, such as enabling notifications or adjusting workflow automation settings.

### Best Practices

* Always label the toggle so the user understands the action they’re taking.
* Use toggles for settings that take effect immediately.
* When a toggle is activated, give users immediate feedback to show if the action was successful.
* Make the component accessible for keyboard navigation and screen readers.

### When Not to Use

* For actions involving more than two options, use other methods instead of a toggle switch.
* Avoid using toggles for actions that require confirmation or have significant consequences.
* Don’t use toggles to make a selection that is better suited for radio buttons, checkboxes, or picklists. 
* Don’t use toggles for irreversible actions (for example, deleting records).
