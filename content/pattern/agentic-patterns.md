Agentic experiences are systems and interfaces that empower users to accomplish their goals with the help of autonomous AI agents. Use these patterns to create agentic experiences in Salesforce products.

## AI-Enabled Cards

Use AI-enabled cards for evergreen AI content within a page. Cards summarize and recommend actions after users generate content, regenerate content, and sometimes automatically generate. AI cards are compact and scannable, while still allowing users to expand AI insights if needed.

An empty agentic cardDoDo show an empty state until a user generates content.An agentic card with a data table and multiple actionsDon’tDon't display complex data on a single card.

Do'sDon'tsShow an empty state until a user generates content.Don't display complex data on a single card.Always show generated content sources.Avoid cluttering dashboards with too many AI-driven cards.Allow for human feedback.Stack cards if more than one is needed.

## Assistant Panels

Assistant panels are optional conversational AI dialogs that users can open to get help and close when the help is no longer needed. Assistant panels facilitate a back-and-forth conversation with predictive and proactive interactions. Users can toggle the visibility of an assistant when needed. Additionally, assistant panels can resize, float, and be pinned above content. When closed, an assistant panel isn't visible but retains its content history and previous state. The prompt composer is docked at the bottom of the assistant panel.

DoDo allow panels in builders to pop out and float in order to provide optimal workspace.Don’tDon’t update data without human consent from the assistant panel.

Do'sDon'tsProvide prompt recommendations based on context and actions.Don’t update data without human consent from the assistant panel.Dock panels within modal dialogs.Don’t automatically open or close the panel without explicit action by the user.Allow panels in builders to pop out and float in order to provide optimal workspace.

## Canvases

Use plain language to ask AI for a custom view of your information. Canvases are fully interactive and customizable areas of content generated by AI. They're also multimodal, personalized, and dynamic. Central to the canvas is a bottom-centered prompt composer.

DoDo display individual cards for each insight requested by the userDoUse multiple input methods besides text, such as voice.

## Disambiguation

Disambiguation provides details and further grounding to AI, which turns vague prompts into data-specific requests.

DoProvide radio button groups with data-grounding options.Don’tDon’t make disambiguation a multi-step process.

Do'sDon'tsProvide radio button groups with data-grounding options.Don’t display checkboxes or text inputs to support disambiguation.Display an action to submit the disambiguation form.Don’t make disambiguation a multistep process.

## Feeds

Actions and prompts appear chronologically in structured feeds. AI and human responses are clearly identified using avatars. The feed history is saved with an option to clear it. Additionally, structured data is presented in cards.

DoDo display feedback on each action.Don’tDon't fill the feed with long text and unnecessary AI responses.

Do'sDon'tsDisplay timestamps and authorship labels for actions.Don’t create dead ends in conversations.Display feedback on each action.Don't fill the feed with long text and unnecessary AI responses.Allow users to refine actions.

## Forms

### AI-Enabled Form Fields

AI can automatically generate form content and provide predictive input suggestions without disrupting manual input. Make sure that users retain control over autofill values in AI-assisted forms.

DoDo indicate AI-enabled fields with a single sparkle icon.DoAlways provide modification controls. Don’t restrict inputs to only generated data without enabling manual overrides.

Do'sDon'tsIndicate AI-enabled fields with a single sparkle icon next to the field label.Don’t enable AI-generated fields if it takes more steps to generate input than manually making a selection.Allow manual overrides on AI-generated entries.Don’t restrict inputs to only generated data without enabling manual overrides.

### Form Dirty State Management

When form fields have unsaved changes, they're in a dirty state. Fields in a dirty state from AI-generated changes are visually and programmatically marked as pending until confirmed. Additionally, AI-generated changes are visually distinct from human-created changes.

DoDo display change indicators for pending AI actions.Don’tAvoid silent AI modifications to critical business records.

Do'sDon'tsDisplay color-coded change indicators for pending AI actions.Don't apply AI-generated changes without proper user validation.Provide undo and confirmation options for AI-suggested modifications.Avoid silent AI modifications to critical business records.

## Human Feedback

Let users share feedback on AI responses. Simple thumbs-up and thumbs-down options are quick. Users can also add detailed feedback in a long-form text input to provide more precise training data.

DoDo place the feedback interaction in a tertiary position so the action content is prioritized.Don’tDon’t provide a single feedback option. Always show both thumbs up and down.

Do'sDon'tsPlace the feedback interaction in a tertiary position so the action content is prioritized.Don’t require long-form text input.Display confirmation after feedback is sent.Don’t provide a single feedback option. Always show both thumbs up and down.

## Prompt Composer

The prompt composer must include text input. Other input types such as voice and other media are optional. Multimodal prompt composers allow for text, voice, and other media types. Prompt composers are always displayed at the bottom center of their parent container.

DoWhile AI is working, change the send button to a stop button.Don’tDon't add prompt input controls outside of the prompt composer.

Do'sDon'tsInclude a send button within the composer.Don't add prompt input controls outside the prompt composer.While AI is working, change the send button to a stop button.Don't move the prompt composer's expected placement.

## Onboarding & Discovery

To effectively onboard new users to an AI experience, show a welcome panel and disclaimer upon their first interaction. Use progressive disclosure to gradually introduce new AI features. Provide first-time users with guided walkthroughs and adaptive tooltips to enhance their understanding. These techniques can ease their transition into using the AI experience.

DoInclude example actions for common tasks to help the user get started.Don’tDon’t expect first-time users to be experts using AI.

Do'sDon'tsInclude example actions for common tasks to help the user get started.Don’t expect first-time users to be experts using AI.Guide users with product-led onboarding.Don’t introduce new features without designing a discovery.
