Margin utility classes provide a quick and consistent way to apply spacing around elements. Margins help define the separation between components, ensuring layouts are visually balanced and aligned with the system's design principles.

**A note on spacing before use:**

- The base unit for all SLDS spacing metrics is **4**.
- Most components have spacing included. These utility classes are for added convenience in laying out components.
- Classes prefixed by `slds-**m**-` are used to add margins to an element. Classes prefixed by `slds-**p**-` are used to add [padding](/develop/padding) to an element.
- Spacing class names use these direction indicators:`top`, `right`, `bottom`, and `left`.
- Use the shortcut `vertical` to apply space to both `top` and `bottom`.
- Use the shortcut `horizontal` to apply space to both `right` and `left`.
- Use the shortcut `around` to apply space to all sides.
- Use the `_xxx-small` through `_xx-large` scale to choose the spacing size needed.
- Use `none` as the size on any of the directions to enforce zero margin.
- For vertical centering, check out our [centered media object](/develop/media-object) instead of applying extra margin or padding.
