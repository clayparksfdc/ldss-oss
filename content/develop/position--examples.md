Position utility classes make it easy to manage the spatial arrangement of components. These utilities use the CSS `position` property to determine how elements are placed within their container or relative to other elements. Use these classes to get more precise control over your layouts.

## Examples

### **Fixed**

`slds-is-fixed` can be used to position a container relative to the viewport.

[https://slds2-metadata-stories-54d63ecb6c29.herokuapp.com/iframe.html?id=utilities-with-code-position--fixed](https://slds2-metadata-stories-54d63ecb6c29.herokuapp.com/iframe.html?id=utilities-with-code-position--fixed)

### Absolute

`slds-is-absolute` can be used to position a container relative to its closest ancestor that's using relative positioning.

[https://slds2-metadata-stories-54d63ecb6c29.herokuapp.com/iframe.html?id=utilities-with-code-position--absolute](https://slds2-metadata-stories-54d63ecb6c29.herokuapp.com/iframe.html?id=utilities-with-code-position--absolute)

### Relative

`slds-is-relative` can be used to contain children if children are absolutely positioned and out of flow. It is also used to position elements without changing the layout.

[https://slds2-metadata-stories-54d63ecb6c29.herokuapp.com/iframe.html?id=utilities-with-code-position--relative](https://slds2-metadata-stories-54d63ecb6c29.herokuapp.com/iframe.html?id=utilities-with-code-position--relative)

### Static

`slds-is-static`resets the positioning of the element back to normal behavior.

[https://slds2-metadata-stories-54d63ecb6c29.herokuapp.com/iframe.html?id=utilities-with-code-position--static](https://slds2-metadata-stories-54d63ecb6c29.herokuapp.com/iframe.html?id=utilities-with-code-position--static)
