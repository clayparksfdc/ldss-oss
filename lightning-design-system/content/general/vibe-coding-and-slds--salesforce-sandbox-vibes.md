Turn a Figma UI design into component code with Agentforce Vibes.

## Use Agentforce Vibes in Your Salesforce Sandbox

Complete the prerequisites. Then follow these steps to create your component code files easily using Agentforce Vibes in your Salesforce sandbox.

### Build Your Code with Agentforce Vibes

Complete these instructions in your sandbox.

ℹ️ Note: As of Spring ‘26, all Salesforce sandboxes include Agentforce Vibes IDE.

![](/assets/images/slds-uploads-tJqPenuiP21QeQy7DJFn1A.png)
*Open a Lightning Experience page.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

Click ⚙️ (Setup).

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

To open Lightning Experience Builder, click Edit Page. Keep this page open—you need it later.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

Open your sandbox in a second tab.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

To launch the Agentforce IDE (formerly Code Builder), click ⚙️ (Setup).

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

Select Agentforce Vibes.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

If this is the first time you’re using Agentforce Vibes, review and accept the Terms & Conditions.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

Click Agentforce Vibes.

  

  
    
      
        
      
      
      The Agentforce avatar.
    
    
  








  
    
    
    
  



  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

If this is your first time using Agentforce Vibes, click I agree to the terms and Enable & Start Building.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

In the Type a message… window, type a prompt to open your Salesforce DX project.Example: Open my Salesforce DX project.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

To build your design, enter a prompt and include a link to your Figma design. For this example, we’re building a checkbox toggle.Example: Grab the currently selected checkbox toggle from Figma by using the Figma MCP server. Build it as a Lightning Web Component (LWC), and use Lightning base components if possible. Then deploy it to my org. Use the Salesforce DX MCP server as necessary. [link to your Figma design].Agentforce Vibes analyzes your Figma design, and begins to create the required files. Don’t worry, this will take a few minutes.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

Pay attention to the Agentforce Vibes chat window. As it builds the files for your Figma design, you may need to interact with it, such as clicking Save, Run Command, or Approve to accept the command and to approve the deployment of the component files as necessary.

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

When complete, you see a message similar to this one:Implemented an SLDS 2-compliant checkbox toggle aligned with the provided Figma design link. The component includes required HTML, JavaScript, XML, and an optional CSS file leveraging SLDS 2 global styling hooks (--slds-g).

  

  
    
      
        
      
      
      
    
    
  








  
    
    
    
  

In your Salesforce sandbox on the open Lightning App Builder page, then drag and drop your new component onto the page.Congratulations! You just built your SLDS 2-compliant component code with Agentforce Vibes.*
